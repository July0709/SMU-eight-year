# 前言
在Github上浏览到[浙江大学](https://github.com/QSCTech/zju-icicles)、[清华大学](https://github.com/Salensoft/thu-cst-cracker)等优质项目，我不禁思考能不能把自己在南医八年的课程资料进行汇总。现在公众号、小红书很发达，甚至有优秀的学长学姐建立了南方医科大学公众号英灵殿。但资料太多，又会让人陷入一种不知从何入手的困境。

这个项目，在记录个人成长历程的同时，整理自己用过的好用的资料，让医学之路不再孤单。

借用詹青云的话：
>我体验过了，我倾尽全力奋斗过了，我对自己算是有交代了，而我那么努力，是为了获得更自由。
>终究这一切，是我们自己所选。

# 平台
为什么采用 GitHub 项目作为平台呢？我有以下一些考虑。

QQ、微信群大多为年级和专业所分隔，无法长期共同地保有；况且群文件也缺乏组织。
GitHub 项目可以使用目录进行文件组织，并且每个目录均可以在显示文件列表的同时显示一个 README，十分适合本项目的需求。
GitHub 带有便捷的 Issue 和 Pull Request 协作功能，并且可以方便地对贡献的质量进行监督和调整。
